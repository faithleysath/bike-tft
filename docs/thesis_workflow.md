# 毕设研究流程与论文映射

这份文档的作用只有一个：

把你的开发流程、实验流程和论文写作流程对齐。

后面你继续推进仓库时，尽量始终遵守下面这个映射关系。

## 使用方式

平时开发时：

- 从 `stages/` 进入当前阶段
- 先看该阶段的 `PLAN.md`
- 做完以后把结果补进 `RESULTS.md`

写毕业论文时：

- 直接按下面的阶段顺序组织章节
- 每个阶段的 `RESULTS.md` 都可以转写成论文章节素材

## 推荐论文结构

1. 绪论与问题定义
   介绍共享单车需求预测和调度问题、研究目标、整体技术路线。
2. 数据来源与阶段 1 基线系统
   说明 Citi Bike 数据、站点小时面板构建方式、TFT MVP 实验链路。
3. 阶段 2 特征增强
   介绍天气、日历、容量/库存近似、POI 等增强特征及其设计原因。
4. 阶段 3 AGCRN 主模型与实验设计
   说明为何把 AGCRN 作为主模型、如何构造图结构、与哪些基线比较、哪些模型机制有效。
5. 阶段 4 调度优化
   把 AGCRN 预测结果转成业务决策，说明缺车/溢车识别和调度任务生成逻辑。
6. 阶段 5 校园迁移与系统展示
   说明数据迁移、AGCRN 迁移、展示界面、应用效果与局限。
7. 总结与展望
   总结研究贡献，并说明未来可扩展方向。

## 阶段产物应该怎么写进论文

每个阶段至少沉淀下面四类材料：

- 目标：这个阶段要解决什么问题
- 方法：用了什么数据、模型或工程流程
- 结果：得到了哪些表、图、指标和结论
- 交接：这一阶段的结论如何支持下一阶段

只要你后续每次都把 `PLAN.md` 和 `RESULTS.md` 补齐，最后写论文时就不会再从零回忆。

## 归档原则

- 原始数据不覆盖
- 中间结果不乱放
- 训练 run 不混阶段
- 每个阶段只认自己目录下的成果
- 阶段完成后，把“代表性 run”写进该阶段 `RESULTS.md`
