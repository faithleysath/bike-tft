# Stage 02 Plan

## 阶段目标

在阶段 1 的 Citi Bike 基线之上，引入关键外部特征，形成增强版需求预测实验。

## 优先级

1. 天气特征
2. 日历 / 节假日特征
3. 站点容量或容量近似
4. 库存近似
5. POI 特征

## 计划任务

- [ ] 确认天气数据源与时间对齐方式
- [ ] 补充日历 / 节假日字段
- [ ] 设计容量或容量近似字段
- [ ] 设计库存近似字段
- [ ] 评估是否接入 POI，或者先保留为可选增强项
- [ ] 生成增强版面板数据
- [ ] 训练增强版 TFT
- [ ] 与阶段 1 基线做初步对比
- [ ] 把结果写入 `RESULTS.md`

## 本阶段输出位置约定

- 原始数据：`data/raw/stage_02_feature_enrichment/`
- 处理结果：`data/processed/stage_02_feature_enrichment/`
- 训练结果：`runs/stage_02_feature_enrichment/`

## 完成标准

- 至少有一版带天气和日历的增强版数据
- 至少有一版增强版 TFT run
- 至少能说明增强特征对结果带来了什么变化
