# Stage 01 Plan

## 阶段目标

用公开的 Citi Bike 数据，把共享单车需求预测主线先做成一个可运行、可复现、可展示的最小版本。

## 完成标准

- 能稳定拿到 Citi Bike 原始订单数据
- 能抽取字段元数据并确认 schema 一致
- 能聚合出站点小时级面板数据
- 能训练一版站点级 TFT 多步预测模型
- 能输出 checkpoint、日志和可视化报告
- 能把当前阶段沉淀成后续阶段的统一起点

## 任务拆分

- [x] 下载 Citi Bike 原始数据
- [x] 检查 12 个月 CSV 的字段结构
- [x] 把订单聚合成 `站点 x 小时` 面板数据
- [x] 生成基础时间特征与站点静态位置特征
- [x] 训练 TFT 基线模型
- [x] 生成中文分析报告
- [x] 把原始数据、处理结果和训练 run 归档到阶段目录
- [x] 为该阶段补齐 `README.md`、`PLAN.md`、`RESULTS.md`

## 本阶段刻意不做的内容

- 天气特征
- POI 特征
- 站点容量
- 实时库存
- 调度优化
- 基线模型对比
- 消融实验
- 校园数据接入

## 阶段结论

阶段 1 的任务不是追求最终精度，而是建立一个可信的起点。

现在这一阶段已经达到“可作为后续研究基线”的标准，可以正式进入阶段 2。

## 留给后续的技术债

下面这些问题不影响阶段 1 作为基线成立，但值得在后续慢慢收掉：

- `build_tft_report.py` 还有静态检查问题
- 当前代表性 run 在较长预测步上存在偏低估倾向
- 当前阶段还没有系统化的基线模型和消融实验
