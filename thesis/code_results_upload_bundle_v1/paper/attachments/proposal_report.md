## 选题的目的和意义

共享单车作为城市短途出行和公共交通“最后一公里”接驳的重要方式，已经成为智慧交通体系中的关键组成部分。但在实际运营中，站点需求会受到天气变化、通勤规律、节假日、学校假期、周边功能区分布以及相邻站点供给状态等多种因素共同影响，呈现出明显的时空异质性与突发波动性。若仅依赖人工经验或静态规则进行车辆调度，容易出现“无车可借”与“无桩可还”的问题，不仅降低用户体验，也会造成调度车辆空驶、运维成本上升和资源配置低效。

本课题旨在面向共享单车运营中的实际问题，构建一套“多源数据融合需求预测—智能再平衡调度—系统可视化展示”的端到端原型系统。课题将融合天气、POI、时间日历、站点网络效应与历史模式等多源外部特征，基于 Temporal Fusion Transformer（TFT）实现多站点、多步长需求预测，并输出 10%、50%、90% 分位数结果，形成面向调度决策的风险感知预测区间；在此基础上，结合当前库存、站点容量和调度成本，进一步设计智能再平衡模块，生成可执行的调度线路和取放方案。

本课题的意义主要体现在以下几个方面：

一是理论意义。共享单车需求预测属于典型的复杂时空序列建模问题，本课题通过引入多源外部特征和可解释的TFT模型，探索静态特征、时变过去特征与已知未来特征在多步长预测中的协同作用，并用分位数回归增强预测的不确定性表达，为时空序列预测与运营决策耦合研究提供参考。

二是工程意义。课题将预测结果直接服务于车辆调度优化，构建“预测驱动决策”的业务闭环，可为共享单车企业在动态投放、再平衡调度、站点规划与运维管理方面提供量化支持。

三是社会意义。通过提高共享单车可用性和调度效率，有助于促进绿色低碳出行、缓解城市拥堵、提升慢行交通系统服务水平，并在一定程度上推动城市交通治理的智慧化和公平化。

## 国内外研究现状

国外在共享出行需求预测方面研究起步较早。早期研究主要采用历史均值、移动平均、ARIMA、Prophet 等传统时间序列方法，这类方法在趋势和季节性建模上具有一定优势，但对共享单车需求中复杂的非线性关系和多因素扰动刻画能力有限[6]。随后，研究者开始引入 XGBoost、LightGBM、随机森林等机器学习方法，以及 LSTM、GRU 等深度学习模型，以提高对天气、节假日、时间周期性等外部特征的建模能力[3]。在概率预测方面，DeepAR 能够输出概率分布形式的预测结果[4]；在深度时序建模方面，N-BEATS 等模型在中长期预测中表现较好[5]。随着 Transformer 架构的发展，基于注意力机制的模型在长序列依赖建模上显示出较强优势[2]。其中，TFT 能同时处理静态特征、历史观测特征和已知未来输入，并具备变量选择与注意力解释能力，已成为多步长时间序列预测的重要代表方法之一[1]。

在共享单车需求研究中，天气、POI、节假日和空间邻接关系等外部特征已被广泛证实对预测精度具有重要作用。许多研究将需求预测对象设置为城市整体需求、区域网格需求或单站点需求，并不断引入空间相关性特征和图结构信息，以增强模型对站点间关联的表达能力。然而，现有研究仍存在若干不足：其一，不少方法更关注点预测，对预测区间和风险不确定性的刻画不足；其二，许多模型虽然提升了预测精度，但可解释性较弱，难以回答“模型为何在某时段、某站点做出该预测”；其三，预测结果与调度决策往往相互割裂，缺乏从需求预测到调度优化的一体化系统设计。

在调度优化方面，车辆再平衡问题通常被抽象为车辆路径问题（VRP）、容量约束车辆路径问题（CVRP）或带时间窗的路径规划问题（VRPTW），常见求解方式包括精确算法、启发式算法和元启发式算法等[10][11]。这些方法能够在一定程度上降低调度成本，但很多研究仍停留在“基于当前库存状态进行反应式调度”的阶段，对未来需求变化考虑不足，容易导致调度滞后。近年来，概念漂移检测[7]、联邦学习[8]和因果推断[9]等方法为时序预测系统的鲁棒性、隐私保护与策略评估提供了新的研究方向，但在共享单车“预测—调度—展示”一体化场景中的应用仍较少。

国内针对共享单车和公共自行车的研究发展较快，已在需求预测、热点区域识别、调度规则设计等方面积累了一定成果，但总体上仍以单模块研究居多，尤其是在多源特征深度融合、预测结果可解释性、分位数区间输出以及预测驱动调度一体化方面仍有较大的提升空间。因此，本课题在现有研究基础上，围绕“多源外部特征融合、可解释多步长预测、智能再平衡调度和系统化展示”展开研究，具有较强的现实针对性和创新价值。

## 主要研究内容

### 1）多源数据获取与特征工程构建

围绕共享单车站点级需求预测任务，收集并整理共享单车历史租还记录、站点基础属性、天气数据、POI 数据、时间日历信息以及相邻站点网络关系等数据。完成数据清洗、缺失值处理、异常值识别、时空对齐、标准化与编码等预处理工作。重点构建以下特征：温度、降水、风速、天气类型等天气特征；站点周边 500m 内餐饮、写字楼、地铁站等 POI 特征；小时、星期、节假日、学校假期、大型活动标志等时间特征；邻近站点库存、流入/流出量、距离矩阵等网络效应特征；过去 7 天同时间段需求、过去 4 周同天需求等历史模式特征。

### 2）基于 TFT 的多步长需求预测模型设计

以站点 ID 作为分组键，构建站点级多步长需求预测任务，预测未来 1—24 小时需求，条件允许时可扩展到网格级预测。按照 TFT 的输入结构组织特征：将站点容量、区域类型、POI 统计等作为静态特征，将历史需求、历史天气、历史库存和流量等作为时变过去特征，将未来小时、是否工作日、节假日日历、天气预报等作为时变未来特征。模型输出采用 0.1、0.5、0.9 分位数，损失函数采用 Pinball Loss，以实现中位预测和区间预测。训练阶段将对窗口长度、预测步长、hidden size、dropout、注意力头数、学习率和 batch size 等参数进行调优，并结合早停与学习率调度提高训练稳定性。

### 3）预测驱动的智能再平衡调度模块设计

基于预测需求与当前库存状态，计算各站点在未来一段时间内的缺车量与溢车量，形成调度任务集合。以最小化调度成本、提升需求满足率为目标，综合考虑站点容量约束、调度车辆装载容量、路径距离与服务优先级，构建共享单车再平衡问题。求解上拟采用两种方案：一是利用 OR-Tools 建立 CVRP 或简化 VRPTW 模型，获得较优调度路径；二是设计“贪心构造+局部搜索”的启发式算法，生成可行调度方案，并与 OR-Tools 结果进行比较分析，以兼顾求解质量和工程可实施性。

### 4）端到端原型系统实现与可视化展示

构建包含数据处理、模型训练、预测推理、调度输出与结果可视化的完整原型系统。数据管道以 Python 脚本和定时流程为基础，条件允许时采用 Prefect 或 Airflow 管理更新过程；空间分析使用 geopandas 处理站点地理数据与距离关系；可视化部分采用 Streamlit 或 Notebook 作为演示前端，并结合 kepler.gl 展示需求热力图、站点缺车/溢车分布和调度路径图。同时输出 TFT 的变量重要性和注意力结果，增强模型可解释性。

### 5）实验设计、对比分析与扩展研究

在预测层面，采用 MAE、RMSE、SMAPE/MAPE、Pinball Loss、区间覆盖率（PICP）等指标评估模型性能；在调度层面，采用缺车率、需求满足率、总调度距离、总调度时间、空驶比例和单位服务提升成本等指标评估调度效果。设置至少 3 个基线模型进行对比，如历史均值/季节性基线、XGBoost/LightGBM、LSTM/GRU 等，并通过消融实验验证天气、POI、网络效应、历史模式等特征模块的贡献。在完成核心任务的基础上，拟尝试加入概念漂移检测，用于监控模型性能下降并触发重训练；若时间允许，可对“增加车辆投放”等运营策略进行简单情景分析，以提升论文的创新性和完整性。

## 实施方案、进度安排及预期效果

### 1）实施方案

本课题拟采用“数据获取与治理—多源特征构建—TFT 预测建模—再平衡调度优化—系统集成展示—实验评估总结”的技术路线实施。首先，优先选取可公开获取的共享单车数据集、气象公开数据、OpenStreetMap POI 数据及节假日日历数据，建立站点级时序样本；若缺少实时库存数据，则通过历史租还记录、站点容量和时间回放构建近似库存序列，以保证研究可行性。其次，在 Python 环境下完成特征工程、模型训练与优化求解，其中预测模块主要基于 PyTorch Forecasting 实现 TFT，空间处理基于 geopandas，调度优化使用 OR-Tools，前端展示采用 Streamlit/Notebook 与地图可视化工具。最后，通过对比实验、消融实验和可解释性分析验证系统有效性，并完成论文与演示系统的整理。

### 2）进度安排

第 1—2 月：完成文献调研，明确研究问题与技术路线；确定数据来源，完成共享单车、天气、POI、节假日等数据的收集、清洗与预处理；初步完成站点级样本构建。

第 3—4 月：完成多源特征工程与基线模型实现；搭建 TFT 模型并开展训练、调参与验证；输出初步预测结果与可解释性分析。

第 5 月：完成预测驱动的再平衡调度模块设计与实现；开展 OR-Tools 和启发式算法实验；完成系统原型界面、热力图与路径图展示；补充消融实验和对比实验。

第 6 月：整理实验结果，完成论文撰写、图表制作和系统优化；补充部署说明、代码注释与演示材料，完成答辩准备。

### 3）预期效果

通过本课题研究，预期形成以下成果：

第一，完成一篇结构完整、论证规范的毕业设计论文，系统阐述多源特征工程、TFT 预测模型、智能调度方法和实验结果。

第二，完成一个可运行的代码工程，具备数据处理、模型训练、预测推理、调度生成和可视化展示等功能，具备较好的复现性。

第三，形成一个可演示的原型系统，能够展示未来需求预测曲线、分位数区间、特征重要性、注意力结果及调度路径方案。

第四，在实验层面，力争使所提出方法在 MAE、RMSE、Pinball Loss 等预测指标上优于基线模型，并在仿真调度场景下有效降低缺车率和无效调度里程，体现预测与调度联动的应用价值。

第五，在创新性方面，形成“多源外部特征融合 + 可解释多步长分位数预测 + 预测驱动调度优化”的完整研究框架，并通过可视化分析和扩展模块增强毕业设计的研究深度。
